# 当日出馬表 CSV のエクスポート手順(お父様向け)

## 用途
予想アプリ(Streamlit Cloud)に毎週末アップロードする「当日の出馬表 CSV」を
TARGET frontier JV から正しい形式で出力する手順。

## 1. 必須エクスポート形式

本アプリは TARGET frontier JV の以下のメニューで作成される CSV を想定:

> **メインメニュー(Z) → 開催成績CSV出力 → フルセット+単勝オッズ**

このメニューで出力された CSV は **52 列のヘッダーなし数値+文字列混在** 形式に
なります。馬名・騎手名・斤量・上3F・通過順位・馬場 等の情報が含まれており、
本アプリの予想ロジック(rating ≥ 100 で ◎本命確定 + 直近5走戦歴マトリクス)が
動作する条件を満たします。

## 2. 手順

1. **TARGET frontier JV** を起動する
2. 上部メニューから **「Z. メインメニュー」** を選択
3. **「開催成績CSV出力」** をクリック
4. 出力形式の選択で **「フルセット+単勝オッズ」** を選ぶ
   ⚠️ **「DC(ダイレクト)」「データカード」等の他メニューは使えません**
5. 当日のレースを範囲指定(全場全レースまとめて OK)
6. CSV ファイルを保存(ファイル名は任意、`.csv` 拡張子)
7. ブラウザで Streamlit Cloud のアプリを開き、左上の
   「当日出馬表 CSV をアップロード」へドラッグ&ドロップ

## 3. うまくいかない場合

### **「この CSV は TARGET frontier JV の DC 系メニューからの出力のようです」**
→ 手順 4 で「DC」「ダイレクト」などのメニューを選んでしまっています。
   **「フルセット+単勝オッズ」** を選び直してください。

### **「列名が想定と異なります」**
→ TARGET のエクスポート設定で列の追加/削除がされている可能性。
   既定設定にリセットして再エクスポートしてください。

### **「CSVの文字コードを判定できませんでした」**
→ 通常 TARGET の出力は Shift_JIS なので問題ないはず。
   万一発生した場合は CSV を Excel 等で開き直し、UTF-8 で保存し直してください。

## 4. なぜ「DC 形式」では使えないのか

DC 形式は TARGET の指数値に特化した圧縮フォーマットで、本アプリが必要とする
以下の情報が **すべて欠落** しています:

- 馬名 / 騎手名(画面に表示する基本情報)
- 斤量(F3 ルール「1600m以上+斤量-3kg」用)
- 単勝オッズ・人気(A1/A2/A4/A5 ワイド候補ルール、B1 減点ルール 用)
- 上3F(C/D/E ルールの主要条件)
- 通過順位(C/D ルール、脚質判定)
- 馬場状態(C/D/E ルールの良/重 分岐)
- 過去走の race_date(F2 休養明け救済の 180 日判定)
- 血統登録番号(過去戦績データベースとの紐付け)

これらが取れないため、◎本命判定も直近5走表示もできません。
必ず「フルセット+単勝オッズ」メニューを使ってください。
